🔌 API Endpoints

Base URL: https://api.devprep.ai/api/v1

Auth — /auth
POST/auth/registerRegister new user
POST/auth/loginLogin, returns access + refresh tokens
POST/auth/logoutInvalidate refresh token🔒 auth
POST/auth/refreshGet new access token via refresh token
GET/auth/verify-email?token=xxxVerify email address
POST/auth/forgot-passwordSend reset email
POST/auth/reset-passwordReset password with token
GET/auth/meGet current user profile🔒 auth
PUT/auth/profileUpdate profile (name, track, avatar)🔒 auth
PUT/auth/change-passwordChange password🔒 auth
Questions — /questions
GET/questionsList questions (filter: category, difficulty, search, page)🔒 auth
GET/questions/:idGet single question with explanation🔒 auth
GET/questions/categoriesGet all categories with question counts🔒 auth
POST/questions/:id/bookmarkToggle bookmark🔒 auth
GET/questions/bookmarksGet user's bookmarked questions🔒 auth
Quizzes — /quizzes
GET/quizzesList available quizzes (filter: type, difficulty)🔒 auth
GET/quizzes/:idGet quiz details + questions🔒 auth
POST/quizzes/:id/startStart quiz session (returns questions without answers)🔒 auth
POST/quizzes/:id/submitSubmit answers, get results + score🔒 auth

🔄 Auth + User Flows

Step-by-step process flows for key user journeys

Register Flow
1
User fills Register form
email, fullName, password, track selection
2
Backend validates
Zod schema validation, check email uniqueness
3
Create user in DB
bcrypt hash password, generate verify_token
4
Send verification email
Nodemailer sends link: /verify-email?token=xxx
5
User clicks link
Backend sets is_verified=true, clears token
✓
Redirect to Login
Registration complete
Login + Token Flow
1
User submits credentials
email + password → POST /auth/login
2
Verify password
bcrypt.compare(), check is_verified
3
Generate tokens
accessToken (15min) + refreshToken (7d), save refresh to DB
4
Return to client
accessToken in response body, refreshToken in httpOnly cookie
5
Axios interceptor
On 401 → auto-call /auth/refresh → retry original request
✓
User authenticated
Store access token in Zustand, redirect to Dashboard
Quiz Flow
1
Browse Quiz List
Filter by type (Frontend/Backend), difficulty
2
Start Quiz
POST /quizzes/:id/start → returns questions (no correct answers)
3
Answer Questions
Timer counts down, user navigates questions, saves answers locally
4
Submit
POST /quizzes/:id/submit → backend calculates score
5
Check Achievements
Auto-check if user earned any new badges, send notification
✓
Result Page
Show score, correct/wrong breakdown, explanations
AI Interview Flow
1
Select Track
Frontend / Backend / Full Stack mode
2
Get Question
Random open-ended question from question bank
3
User writes answer
Textarea with timer (optional)
4
POST /ai/evaluate
Send question + user_answer to backend
5
Claude API call
Structured prompt → returns JSON: score, feedback, ideal_answer, resources
✓
Show Results
Score card, mistake explanation, ideal answer, learning links. Save to AISession
